Inline-Elemente

Das ist ein Text mit einem Span-Element [span]. Manchmal ist wichtig, Teile des Textes stark zu betonen [strong] oder auch auch nur leicht hervorheben[em]. Rein stilistische Änderungen sind zwar ohne besondere Betonung [b], als andere Darstellung[i], ›unartikuliert‹ [u] möglich. Da sie keine semantische Bedeutung tragen, sollten diese stilistischenÄnderungen per CSS umgesetzt werden und nicht im HTML. Unterstreichungen sind als starke typografische Konvention im Web für Links [a] vorbehalten.

Änderungen im Text können markiert [del] nachvollziehbar gemacht [ins] werden. Veralteter Text kann als nicht mehr relevant markiert werden [s]. Allerdings wird das von Screenreadern ignoriert.

Brech die Zeile nur
wenn’s ein Gedicht oder
eine Post-Adress’ wie in
24943 Flensburg ist. [br]

In manchen Schriftsystem muss man von rechts nach links lesen, مرحبا [bdi]. Dann kann es nötig sein, die normale Richtung umzudrehen. [bdo]

Für Formeln wie H2O [sub] oder bestimmte Kennzeichnungen sind Superscript®[sup] oder Subscript© nötig. Das Kleingedruckte soll oft nicht gelesen werden, [small], ist aber wichtig.

HTML [abbr] ist eine Abkürzung. Ein HTML-Validator [dfn] ist per Definition ein Programm, was den Syntax überprüft. Denn onyl valid HTML is good HTML [q] sagte schon Tim Berners-Lee [cite] in einem kurzen Zitat mit unklarer Quelle.

Das gesuchte Wort sollte in Suchergebnissen markiert [mark] werden.

Variablen wie in x = y - 2 [var] können Werte speichern. Der entsprechende Elixir-Code x=y+2 [code] kann mit iex -e 'x=y+2' [kbd] in der REPL getestet werden. Mit der Fehlermeldung error: undefined variable "y" zeigt der Elixir-Compiler das Problem an.
